Как задеплоить агента на клиентские компы

Шаг 1 — Один раз: сборка на твоей машине (нужен Python)

build.bat

Выдаёт два файла в dist\:
- srp-setup.exe — всё в одном, Python не нужен. Это то, что передаёшь на клиентские компы.
- srp-agent.exe — агент без установщика (для GPO/ручного разворачивания)

---
Шаг 2 — На каждом клиентском компе

Скопируй srp-setup.exe и запусти от Администратора:

srp-setup.exe --server http://192.168.1.10:8000

Или двойной клик — спросит URL интерактивно.

Что делает:
1. Распаковывает srp-agent.exe → C:\SRP\
2. Пишет C:\SRP\config.json с твоим URL сервера
3. Регистрирует задачу Windows (SYSTEM, запуск при загрузке, перезапуск при сбое × 3)
4. Стартует агента сразу — ребут не нужен

Удаление:
srp-setup.exe --uninstall

---
Файлы на клиентском компе после установки

C:\SRP\
    srp-agent.exe   — бинарь агента
    config.json     — URL сервера + device_id (сохраняется при обновлениях)
    srp-agent.log   — лог (ротируется)

---
GPO / mass-deploy

srp-setup.exe --server http://192.168.1.10:8000 --token mysecret

Тихий режим (нет промптов, exitcode 0/1) — подходит для Group Policy Startup Script или любого RMM.
---------------------------------------------
Паролей два, они разные

1. ingest_token — токен агент→сервер

Агент при каждой отправке добавляет заголовок X-SRP-Token: <token>. Сервер проверяет. Если не совпадает — 401.

По умолчанию: пустой = аутентификация выключена.

Установить при деплое:
srp-setup.exe --server http://192.168.1.10:8000 --token mysecret

Изменить на уже установленном компе — отредактировать C:\SRP\config.json:
"ingest_token": "mysecret"

На сервере — в ServerConfig (или переменная окружения при запуске):
# server/config.py
ingest_token: str = "mysecret"
Или через env:
SRP_INGEST_TOKEN=mysecret python -m server.main

---
2. config_password_hash — пароль на изменение config.json

Защищает конфиг на клиентском компе от ручной правки. Хранится как PBKDF2-SHA256 (plaintext никогда не пишется). Агент всегда стартует без пароля — он нужен только при сохранении изменений.

По умолчанию: пустой = защиты нет.

Инфраструктура написана в client/config.py (hash_password / verify_password / save_config), но CLI-команды для установки ещё нет в agent.py — это заглушка под будущее.

---
3. Дашборд (/)

Аутентификации нет вообще. Открыт всем на сети. Если нужно — стандартное решение: nginx reverse proxy с basic auth перед портом 8000.

---
Итого для продакшна: обязательно задай ingest_token одинаковый на сервере и через --token при деплое агентов. Дашборд пока открытый — держи порт 8000 в закрытом сегменте сети.
