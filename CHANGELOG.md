## [Unreleased]

### Added

- `CLAUDE.md` — гид для разработки: команды, жёсткие инварианты, обязательное
  применение skills и субагентов.
- Хук `.claude/hooks/ruff_format.py` (PostToolUse) — авто `ruff check --fix` +
  `ruff format` для изменённых `.py`.
- Память проекта `.claude/memory/` — индекс `MEMORY.md` и заметки об инвариантах.
- Этот `CHANGELOG.md`.
- Слой доверия телеметрии (`server/trust/`): классификация источников
  `state` (гейт) + `weight` (модуляция), collector⊥semantic, объяснимое
  разрешение доверия по доменам (контракт — `telemetry-trust-contract.md`).
- Агент сообщает здоровье каждого источника телеметрии (`source_health`) в каждом
  конверте — статус сбора (`ok/partial/empty/timeout/blocked/absent`) по каждому
  источнику; сервер сможет отличать «источник недоступен/пуст» от «здоров»
  (недоступность ≠ здоровье).
- Сервер вычисляет доверие по источникам/доменам на каждом приёме (lineage) и
  размечает им риск: класс отказа с недостоверным доменом показывается как
  UNKNOWN (без ложно-уверенной вероятности); устройство с недостоверной
  идентичностью помечается `untrusted`, а его day-1 оценки не показываются (§7).
- Обнаружение «пропавших» источников: источник, который раньше отдавал данные, а
  теперь не отвечает (`empty/timeout/blocked/absent`), помечается `regressed` и
  выводится баннером на дашборде — отличается от никогда не виденного источника.
- Привязка устройства к объекту/организации: вручную присваиваемый `site_code`
  (+ опц. `site_name`) приходит в конверте и хранится на устройстве; флот
  группируется по объекту (масштабирование на множество фирм/площадок).

## [0.1.0] - 2026-05-29

Первый MVP системы раннего предупреждения отказов.

### Added

- Тонкий агент (`client/`) на чистом стандартном Python: сбор `inventory`,
  `historical`, `heartbeat`, `events` через PowerShell; отправка в общем
  конверте с офлайн-буфером (`buffer.jsonl`, FIFO-досылка).
- Сервер (`server/`) на FastAPI + SQLite: приём `/api/v1/ingest`, хранение
  истории, пересчёт оценок на каждое сообщение.
- 4 оценки «день-1» (`performance`, `reliability`, `wear`, `risk_exposure`) и
  байесовский риск по классам отказа в лог-оддсах с объяснимым вкладом признаков.
- Веб-дашборд парка и устройства (серверный рендер Jinja2, autoescape вкл.).
- Контракт сообщений — единый источник истины (`shared/schema.py`, pydantic v2).
- Качество кода: `ruff`, `mypy`, `bandit`, `pytest` (порог покрытия 80 %),
  `pre-commit`, GitHub Actions CI, `Makefile`, `smoke.py`.

### Security

- Серийные номера дисков хешируются (SHA-256) в агенте; сырой серийник не
  покидает процесс.
- Jinja2 autoescape включён; весь SQL параметризован; вход валидируется на
  границе через pydantic.
