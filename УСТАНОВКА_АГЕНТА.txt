==========================================
  УСТАНОВКА АГЕНТА SRP — быстро и просто
==========================================

Что это: маленькая программа, которая собирает данные о компьютере
и отправляет их на сервер. Ставится за пару минут.


ШАГ 0. Что нужно один раз
-------------------------
На компьютере должен стоять Python 3.9 или новее.Проверить: открой "Командную строку" (cmd) и набери:

    python --version

Если показывает версию (например Python 3.10.11) — всё ок, иди дальше.
Если пишет ошибку — поставь Python с https://python.org
(при установке поставь галочку "Add Python to PATH").


ШАГ 1. Скопировать папку к себе
-------------------------------
Зайди в сетевую папку с агентом, например:

    \\СЕРВЕР\share\srp

Скопируй её целиком на локальный диск, например в:

    C:\srp

Важно: запускать НЕ из сетевой папки, а со своего диска.


ШАГ 2. Запустить и указать адрес сервера
----------------------------------------
В папке C:\srp найди файл:

    agent_ip.bat

Запусти его двойным щелчком. Он спросит адрес сервера — введи его.

Пример (если знаешь только IP):

    192.168.1.10

Пример (если знаешь полный адрес):

    http://192.168.1.10:8000

Нажми Enter. Всё — агент работает и сам перезапускается, если что.
Окно закрывать не надо, пусть висит.


ГОТОВО.
=======

Если адрес сервера всегда один и тот же и менять его не надо —
можно вместо этого запускать start_agent.bat (адрес уже прописан
внутри настроек). Но для первого раза проще agent_ip.bat.


Если что-то не так
------------------
- "python не является внутренней командой" — Python не установлен
  или не добавлен в PATH (см. ШАГ 0).
- "server_url is not set" — запусти agent_ip.bat и введи адрес.
- Агент пишет ошибки сети — проверь, что сервер включён и адрес верный
  (например, открой в браузере http://192.168.1.10:8000 ).


==========================================
  ДЛЯ ИНЖЕНЕРОВ — параметры командной строки
==========================================

Запуск:  python -m client.agent [параметры]

  --server URL     адрес сервера, перебивает server_url из config.json
  --once           один проход всех коллекторов и выход (без цикла)
  --log-file PATH  дублировать лог в файл (ротация); для службы
  --verbose        debug-логирование

Примеры:

  # разовая проверка связи с сервером, подробный лог
  python -m client.agent --once --server http://192.168.1.10:8000 --verbose

  # постоянная работа с логом в файл
  python -m client.agent --server http://192.168.1.10:8000 --log-file C:\srp\agent.log

  # server_url уже в config.json — параметры не нужны
  python -m client.agent

Коды выхода:  0 ок · 2 не задан server_url.


==========================================
  САМЫЙ БЫСТРЫЙ ЗАПУСК ИЗ СЕТЕВОЙ ПАПКИ
==========================================

ВАЖНО: cmd НЕ умеет делать UNC-путь рабочим каталогом, поэтому
bat-файлы с `cd /d "%~dp0"` запускать прямо с \\СЕРВЕР\... нельзя
(будет "CMD does not support UNC paths"). Решение — pushd или копия.

Вариант А (Python, одна строка — копирует и запускает):

  robocopy \\СЕРВЕР\share\srp C:\srp /E /NJH /NJS & C:\srp\agent_ip.bat

Вариант Б (служба, exe-сборка, ставится с боота + рестарт при падении):
  правый клик по install.bat в сетевой папке → "Запуск от имени
  администратора". install.bat НЕ использует cd, работает с UNC напрямую:
  копирует srp-agent.exe + config.json в C:\SRP и регистрирует
  задачу планировщика (SYSTEM, старт при загрузке, рестарт x3).
  Требует собранный srp-agent.exe рядом с install.bat.

Вариант В (служба через PowerShell, задаёт server_url сразу):

  powershell -NoProfile -ExecutionPolicy Bypass -File ^
    \\СЕРВЕР\share\srp\client\deploy\install-service.ps1 ^
    -ServerUrl http://192.168.1.10:8000

Рекомендация: массовая раскатка → Вариант Б/В (служба, переживает
ребут). Разовый/тестовый запуск → Вариант А.


==========================================
  ПОЛНЫЙ СПРАВОЧНИК КЛЮЧЕЙ
==========================================

--- 1. Агент: python -m client.agent [ключи] ---

  --server URL     адрес сервера; перебивает server_url из config.json
  --once           один проход коллекторов и выход (без бесконечного цикла)
  --log-file PATH  дублировать лог в файл с ротацией
  --verbose        debug-логирование (по умолчанию INFO)
  -h, --help       показать справку и выйти

Это ВСЕ ключи агента. Остальная настройка — через config.json.

--- 2. Установщик службы: install-service.ps1 [ключи] ---
    (запускать из PowerShell ОТ АДМИНА)

  -ServerUrl URL     ОБЯЗАТЕЛЬНЫЙ. Адрес сервера (http:// или https://)
  -SiteCode CODE     код площадки/группы (W1.1), необязательно
  -SiteName NAME     читаемое имя площадки, необязательно
  -IngestToken TOK   общий токен приёма (должен совпадать с сервером).
                     ВНИМАНИЕ: в командной строке токен виден в списке
                     процессов/истории — на важных хостах лучше править
                     config.json напрямую
  -PythonExe PATH    путь к python.exe; по умолчанию первый python в PATH
  -TaskName NAME     имя задачи планировщика; по умолчанию "SRP Agent"

--- 3. Ключи config.json (правятся вручную в файле) ---

  server_url               адрес сервера
  device_id                ID хоста (ставится сам при первом запуске —
                           НЕ трогать, не копировать между машинами)
  inventory_interval_sec   период инвентаризации (по умолч. 86400)
  historical_interval_sec  период исторических данных (86400)
  heartbeat_interval_sec   период heartbeat (300)
  events_interval_sec      период сбора событий (900)
  print_interval_sec       период сбора печати (900)
  http_timeout_sec         таймаут HTTP-запроса (15)
  buffer_path              файл буфера при недоступности сервера
  site_code / site_name    код и имя площадки
  org_code / dept_code     код организации и отдела
  comment                  произвольный комментарий
  ingest_token             токен приёма (должен совпадать с сервером)
  offline_mode             true = работать без сервера (только локально)
  config_password_hash     хэш пароля защиты настроек
  agent_version            версия агента
  update_channel           канал обновлений (stable)


==========================================
  ЕЩЁ ПРИМЕРЫ УСТАНОВКИ
==========================================

# 1. Самый быстрый тест на одном компе (сервер+агент, два окна)
run_local.bat

# 2. Разовая проверка связи, ничего не ставя
python -m client.agent --once --server http://192.168.1.10:8000 --verbose

# 3. Постоянная работа вручную, лог в файл
python -m client.agent --server http://192.168.1.10:8000 --log-file C:\srp\agent.log

# 4. Установка службой (минимум) — из PowerShell от админа
powershell -ExecutionPolicy Bypass -File C:\srp\client\deploy\install-service.ps1 ^
  -ServerUrl http://192.168.1.10:8000

# 5. Установка службой с площадкой и токеном
powershell -ExecutionPolicy Bypass -File C:\srp\client\deploy\install-service.ps1 ^
  -ServerUrl http://192.168.1.10:8000 -SiteCode MSK -SiteName "Москва офис" ^
  -IngestToken s3cr3t-token

# 6. Установка службой с нестандартным Python и именем задачи
powershell -ExecutionPolicy Bypass -File C:\srp\client\deploy\install-service.ps1 ^
  -ServerUrl http://192.168.1.10:8000 ^
  -PythonExe "C:\Python310\python.exe" -TaskName "SRP Agent Prod"

# 7. Удалить службу
powershell -ExecutionPolicy Bypass -File C:\srp\client\deploy\uninstall-service.ps1

# 8. Запуск из сетевой папки одной строкой (копия + старт)
robocopy \\СЕРВЕР\share\srp C:\srp /E /NJH /NJS & C:\srp\agent_ip.bat

# 9. Offline-режим (без сервера): в config.json выставить
#    "offline_mode": true  — затем
python -m client.agent
