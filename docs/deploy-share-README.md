# SRP — установка одной командой (сетевая папка)

Установка агента + трея на ПК **одной командой, без вопросов** (трей-спека §6).
Старый onefile-установщик (`installer/`) заменён этим onedir-механизмом.

## Раскладка скрытой шары `\\server\srp$\`
```
setup.exe                ← установщик (манифест UAC-admin, поднимает права сам)
config.template.json     ← политика организации (интервалы, tray_cert_warn_days,
                            tray_notify_hours, helpdesk_contact, site_code) — можно
                            править прямо на шаре, без пересборки
VERSION                  ← маркер версии для идемпотентного апгрейда
install-101-7.bat        ← готовые BAT по подразделениям (опц., генератор — этап 7)
payload\                 ← srp-agent.exe, srp-tray.exe, DLL, .ico, task_template.xml
```
> Шара **скрытая** (`$`), ACL — **только группа ИТ**: в BAT лежат пароль/токен.

## Сборка (на машине разработчика, один раз на релиз)
```bat
build.bat
```
PyInstaller (`requirements-build.txt`, **только для сборки** — `client/` остаётся
чистым stdlib) собирает три артефакта в `dist\share\`; скопируйте их на шару.

## Установка ПК (BAT двойным кликом → UAC → готово)
```bat
\\server\srp$\setup.exe --server http://192.168.1.10:8000 ^
  --org 101 --dept 7 --password "Zx9!..." --token "mysecret"
```
| Параметр | Обяз. | Назначение |
|---|---|---|
| `--server URL` | да* | адрес сервера (*может прийти из config.template.json) |
| `--org CODE` | да | код организации, `^[A-Za-z0-9_-]{1,16}$` |
| `--dept CODE` | нет | код подразделения, тот же формат |
| `--password X` | нет | пароль трея/конфига → PBKDF2-хеш (в открытом виде не хранится) |
| `--token X` | нет | ingest_token (заголовок X-SRP-Token) |
| `--helpdesk "…"` | нет | контакт поддержки (перекрывает шаблон) |
| `--comment "…"` | нет | свободная метка машины |
| `--no-tray` | нет | без иконки (серверы/киоски — Run-ключ не ставится) |
| `--allow-offline` | нет | ставить, даже если сервер сейчас недоступен |
| `--quiet` | авто | включается сам, когда заданы `--server` и `--org` |
| `--uninstall [--purge]` | — | удаление (+ данные/логи по `--purge`) |

Заданы обязательные → тихий режим автоматически. Запуск `setup.exe` совсем без
параметров — интерактив (спросит сервер и орг-код).

## Что делает установщик (< 60 c)
1. UAC (манифест) → 2. `robocopy payload\ C:\SRP\` → 3. **ACL** `icacls C:\SRP`
(снимает наследование, даёт запись только SYSTEM/Администраторам — настоящая защита
`config.json`) → 4. `config.json` = merge(шаблон + параметры), `device_id` и прежние
значения сохраняются, UTF-8 **без BOM** → 5. журнал печати `wevtutil … /e:true`
(best-effort) → 6. валидация `srp-agent.exe --once` (провал → код 3; смягчает
`--allow-offline`) → 7. задача планировщика из `task_template.xml` (AtStartup, SYSTEM,
Highest, рестарт 3×1мин) + Run-ключ трея (кроме `--no-tray`) → 8. старт.

## Коды возврата (для RMM / GPO)
| код | смысл |
|---|---|
| 0 | успех |
| 2 | неверные/неполные параметры |
| 3 | сервер недоступен (без `--allow-offline`) |
| 4 | копирование/ACL |
| 5 | регистрация задачи/автозапуска |
| 1 | прочее |

Полный протокол каждой установки/апгрейда — `C:\SRP\install.log` (append).
Массовый раскат: `psexec \\PC -s \\server\srp$\install-101-7.bat`, GPO Startup
Script, любой RMM — итог виден по коду возврата.

## Обновление и удаление
- **Апгрейд** = тот же BAT с новой версией в шаре (стоп → robocopy дельта → старт;
  `config.json` не трогается).
- **Удаление**: `setup.exe --uninstall` (стоп задачи, taskkill трея, удалить задачу +
  Run-ключ; файлы/данные — по `--purge`).

## Безопасность
Пароль/токен лежат в BAT на шаре и мелькают в списке процессов. Митигация: шара
**скрытая** + ACL «только ИТ» (это деплой-инструмент, не публичная папка). На особо
чувствительных хостах — ставить без `--password/--token` и вписать их в `config.json`
вручную.
