# SRP — сборка и установка агента

Без воды: как запустить локально, собрать релиз и установить агента на ПК.
Все ключи установщика — в §3.

## 1. Локальный запуск (разработка)

| Файл | Что делает |
|---|---|
| `start_server.bat` | сервер на `http://localhost:8000` (освобождает порт, авто-рестарт) |
| `start_agent.bat [URL]` | агент постоянно (авто-рестарт). Без аргумента — сервер из `client\config.json`; с аргументом — `start_agent.bat http://192.168.1.10:8000` |
| `agent_ip.bat` | то же, но спрашивает адрес сервера при запуске |
| `run_local.bat` | быстрый тест: сервер + один проход агента в двух окнах |
| `build.bat` | сборка релиза (см. §2) |

Дашборд: `http://localhost:8000` · API: `http://localhost:8000/api/v1/scores`

## Продакшн: TLS

Сервер SRP слушает HTTP; в проде ставьте перед ним reverse-proxy с TLS
(caddy — две строки конфига, или nginx+certbot) и указывайте агентам
`https://…` в `server_url`. Собственный PKI не заводим сознательно
(cctodo §1, анти-цели): токен ingest + TLS прокси закрывают модель угроз
«внешний злоумышленник в сети».

## 2. Сборка релиза (один раз на версию, на dev-машине)

```bat
build.bat
```

PyInstaller (зависимость **только для сборки** — `requirements-build.txt`; код
агента остаётся чистым stdlib) собирает в `dist\share\`:

- `setup.exe` — установщик (манифест UAC-admin, поднимает права сам)
- `payload\` — `srp-agent.exe`, `srp-tray.exe`, DLL, `.ico`, `task_template.xml`
- `config.template.json` — политика организации · `VERSION` — маркер версии

Плюс `dist\updates\srp-agent-update-<ver>.zip` + `manifest.json` — пакет
автообновления парка (см. «Обновление и удаление» ниже).

Скопируйте содержимое `dist\share\` в скрытую папку-шару `\\server\srp$\`
(ACL — только группа ИТ: в BAT лежат секреты).

## 3. Установка на ПК (одна команда)

Техник запускает BAT своего подразделения (двойной клик → UAC):

```bat
\\server\srp$\setup.exe --server http://192.168.1.10:8000 --org 101 --dept 7 --password "Zx9!" --token "mysecret"
```

Готовый BAT с подставленными кодами организаций можно получить на странице
дашборда **«развёртывание»** (`/deploy`) — секреты там остаются плейсхолдерами.

### Все ключи

| Ключ | Обяз. | Назначение |
|---|---|---|
| `--server URL` | да* | адрес сервера. *Можно опустить, если он задан в `config.template.json` |
| `--org КОД` | да | код организации, формат `^[A-Za-z0-9_-]{1,16}$` |
| `--dept КОД` | нет | код подразделения, тот же формат |
| `--password X` | нет | пароль трея/настроек → хранится как PBKDF2-хеш (в открытом виде не пишется) |
| `--token X` | нет | `ingest_token` (заголовок `X-SRP-Token`, как на сервере) |
| `--helpdesk "…"` | нет | контакт поддержки (перекрывает шаблон) |
| `--comment "…"` | нет | свободная метка машины |
| `--no-tray` | нет | без трей-иконки (серверы/киоски — Run-ключ не ставится) |
| `--allow-offline` | нет | ставить, даже если сервер сейчас недоступен (агент буферизует) |
| `--quiet` | авто | тихий режим; включается сам, когда заданы `--server` и `--org` |
| `--uninstall` | — | удалить: стоп задачи, taskkill трея, снять задачу + Run-ключ |
| `--purge` | — | с `--uninstall`: ещё и удалить `C:\SRP` (данные/логи) |

Без параметров `setup.exe` спросит сервер и код организации интерактивно.

### Что делает установщик

Копирует в `C:\SRP` → закрывает права (`icacls`: запись только
SYSTEM/администраторам — настоящая защита `config.json`, не пароль) → пишет
`config.json` (UTF-8 без BOM, `device_id` сохраняется) → включает журнал печати →
проверяет связь одним проходом → ставит задачу планировщика (SYSTEM, при старте,
рестарт 3×1 мин) + автозапуск трея → стартует.

### Коды возврата (для RMM/GPO)

`0` ок · `2` неверные/неполные параметры · `3` сервер недоступен (без
`--allow-offline`) · `4` копирование/ACL · `5` регистрация задачи/автозапуска ·
`1` прочее. Полный протокол — `C:\SRP\install.log`.

### Обновление и удаление

- **Автообновление (основной путь)**: агент раз в час спрашивает сервер о
  новой версии. Выкладка: `build.bat` → в `dist\updates\` появляются
  `srp-agent-update-<ver>.zip` + `manifest.json` → скопировать ОБА файла в
  `server\updates\` на сервере → парк обновится сам (скачивание с проверкой
  sha256 и HMAC на `ingest_token`, применение через одноразовую SYSTEM-задачу
  «SRP Agent Update», `config.json` не трогается, трей поднимется при
  следующем входе). Версии и статус видны на дашборде: Флот, карточка
  устройства, `/deploy`.
- **Ручное обновление (fallback)**: тот же BAT с новой версией в шаре (стоп →
  копирование дельты → старт; `config.json` не трогается).
- **Удаление**: `setup.exe --uninstall` (вместе с данными — `--purge`).

Раскладка шары — `deploy-share-README.md`.
