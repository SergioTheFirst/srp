# Сеть — фаза 3: активный поиск устройств (скан за гейтом)

**Дата:** 2026-06-11
**Статус:** дизайн (производный от `2026-06-10-network-tools-design.md` §8, фаза 3).
**Реализация ЗА ГЕЙТОМ:** не начинается без письменного разрешения безопасности владельца сети
(активный скан выглядит как атака для EDR/IDS). Эта спека — только дизайн.
**Объём:** агент + контракт (аддитивно) + минимальный показ на сервере.

---

## 1. Цель

Фазы 1–2 видят только то, что машины уже знают (ARP). Устройство, с которым давно
не общались, в ARP отсутствует → его нет на карте. Фаза 3 закрывает дыру:
**ping-перебор СВОЕЙ /24-подсети** наполняет ARP-таблицу, существующий конвейер
(Ф1 соседи → Ф2 карта) автоматически показывает найденное. Плюс **проверка
фиксированного набора портов** у ответивших — подсказка типа устройства
(9100 = принтер, 445 = Windows, 3389 = RDP, 80/443 = веб-морда).

## 2. Гейт безопасности (главное решение)

| # | Решение | Почему |
|---|---------|--------|
| G1 | **Выключен по умолчанию**: `ClientConfig.network_scan_enabled = False`; включение = явная правка config.json на конкретной машине | Активный скан выглядит как атака для EDR/IDS; включать можно только там, где есть письменное разрешение владельца сети — это ответственность оператора, агент лишь даёт возможность |
| G2 | **Цели и порты НЕ конфигурируются**: только /24 вокруг собственного RFC1918-адреса; порты — фиксированный список в коде (80, 443, 445, 3389, 9100) | Конфигурируемые цели/порты превращают агента в инструмент атаки; фиксированный неизменяемый список — в инструмент инвентаризации. Сервер НЕ может удалённо менять цели (анти-цель «remote agent control» из W1) |
| G3 | **Rate-limit зашит в код**: пробы строго последовательно, пауза ≥200 мс между адресами, TCP-таймаут ≤500 мс/порт, общий бюджет прохода ≤15 мин; при превышении — остановка с пометкой `partial` | Медленный одиночный обход не похож на flood; ограничитель не в конфиге — оператор не может «ускорить» |
| G4 | **Каденс — не чаще 1 раза в 24 ч** (state-файл по образцу `print_state.json` хранит время последнего прохода); сбой агента/рестарт не сбрасывает таймер | Карта меняется медленно; частый скан = шум для IDS и сети |
| G5 | **Сканируется ТОЛЬКО своя подсеть /24** и только если собственный адрес RFC1918; APIPA/публичный адрес → скан не выполняется, источник `blocked` с причиной | Чужие/публичные диапазоны — никогда; согласуется с инвариантом «только RFC1918 покидает агента» |
| G6 | **Никакого контента**: только факт ответа ICMP + факт открытости TCP-порта (connect → сразу close). Без баннер-граббинга, без чтения данных из сокета | Подсказка типа устройства не требует данных; чтение баннеров = снятие отпечатков, другой класс риска |

## 3. Что входит и НЕ входит

**Входит:**
- Конфиг-флаг `network_scan_enabled` (default False) + валидация.
- Новый под-сбор в коллекторе network (или отдельный `client/collectors/netscan.py`):
  ping-перебор /24 + TCP-проба фиксированных портов у ответивших.
- Контракт: аддитивное необязательное поле в `HistoricalPayload` (без смены `CONTRACT_VERSION`).
- Сервер: порт-подсказки на карте Ф2 («похоже на принтер: 9100») — read-side only (D7).
- Тесты + строка CHANGELOG + предупреждение в README/install-доках о требовании письменного разрешения.

**НЕ входит (анти-цели):**
- Конфигурируемые цели/порты/скорость; диапазоны шире своей /24; IPv6-перебор.
- Баннер-граббинг, версия сервиса, ОС-fingerprinting, UDP-сканирование.
- Влияние на скоры здоровья/доверия: скан — инвентаризация, не сигнал здоровья
  (`network_scan` в `DOMAIN_SOURCES` НЕ добавляется; recompute не трогается).
- Удалённое включение скана с сервера (только локальный config.json).

## 4. Контракт — `shared/schema.py` (аддитивно)

- `NetScanFinding(_Base)`: `ip: str`, `open_ports: list[int] = []` (только из фиксированного списка).
  MAC не дублируем — после наполнения ARP сосед уже приходит в `network_neighbors` с MAC.
- `NetScanInfo(_Base)`: `scanned_at: float`, `subnet: str` (вида `192.168.1.0/24`),
  `hosts_probed: int`, `hosts_alive: int`, `findings: list[NetScanFinding]` (cap `NET_SCAN_FINDINGS_MAX = 256`),
  `truncated: bool` (бюджет G3 исчерпан).
- `HistoricalPayload.network_scan: Optional[NetScanInfo] = None` — поле есть только когда скан включён
  И проход был в этом цикле. `CONTRACT_VERSION` не растёт.

## 5. Агент

- **Ping:** `Test-Connection -Count 1 -Quiet` пакетами адресов (WinPS 5.1: БЕЗ `-TimeoutSeconds` —
  его нет в 5.1; бюджет времени держим размером пакета + паузами, `[[agent-powershell-51-floor]]`).
- **Порты:** чистая stdlib — `socket.create_connection((ip, port), timeout=0.5)` → close.
  Без PowerShell на этом шаге; никакой записи/чтения в сокет (G6).
- **Состояние:** `client/netscan_state.json` (рядом с `print_state.json`): `last_scan_ts`.
  Каденс проверяется ДО запуска; рестарт агента не обнуляет.
- **source_health:** ключ `network_scan`: выключен → источника нет вообще (не «blocked» — это выбор,
  не сбой); включён и прошёл → `ok`/`partial` (бюджет исчерпан); адрес не-RFC1918/APIPA → `blocked`
  с причиной. Никакого тихого None.
- Сбой любого шага → пустой блок + статус, остальной historical уходит как обычно.

## 6. Сервер

- Хранение: payload `historical` уже JSON-блоб — `network_scan` сохраняется автоматически; новая таблица не нужна.
- `server/analytics/netmap.py`: узел с открытым портом получает RU-подсказку типа
  (9100 → «похоже на принтер», 3389 → «RDP включён», 445 → «Windows-узел», 80/443 → «веб-интерфейс»).
  Read-side only; в `recompute_scores` НЕ входит (урок W4.0/D7).
- Доверие: `evaluate_trust` фиксирует `network_scan` как collector-only источник без домена и без
  семантического валидатора (паттерн `network` Ф1); скоры дня-1 не меняются (тест пинит).
- Дашборд `/netmap`: бейдж «найден сканом» у узлов, которых нет в пассивном ARP других агентов.

## 7. Тесты

- Гейт-инварианты (главные): default выключен; APIPA/публичный адрес → `blocked`, проб нет;
  каденс <24 ч → прохода нет; список портов = ровно {80, 443, 445, 3389, 9100} (пин против расширения);
  бюджет исчерпан → `truncated=True` + `partial`.
- Порт-проба: open/closed/timeout (socket замокан); в сокет ничего не пишется и не читается.
- Контракт: конверт без `network_scan` валиден (старый агент); round-trip; версия не растёт; cap findings → 422.
- Сервер: ingest со сканом → подсказки на карте; скоры/доверие дня-1 не изменились; `network_scan`
  не появился в `DOMAIN_SOURCES`.
- mypy на `client/` чистый; PS-сниппеты — только конструкции 5.1.

## 8. Открытые вопросы (решить при реализации, ПОСЛЕ снятия гейта)

1. Размер ping-пакета (адресов на один вызов `Test-Connection`) vs паузы — подобрать так, чтобы
   полный /24 укладывался в бюджет G3 с запасом.
2. Куда класть порт-пробы: внутрь `collect_network()` (один источник) или отдельный коллектор —
   решает читаемость `network.py` (<800 строк, функции <50).
3. Нужен ли отдельный бейдж «сосед появился ПОСЛЕ скана» (пассивный ARP до/после) или достаточно
   `findings` — смотреть по реальному снимку.
