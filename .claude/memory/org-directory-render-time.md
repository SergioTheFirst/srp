# Справочник орг/отделов: файл на сервере, имена только render-time

**Решение (2026-06-12, спека tray-client §7):** коды организации/подразделения
расшифровываются в полные названия из `org_directory.json` (путь в ServerConfig /
env SRP_ORG_DIRECTORY; mtime-перечитывание без рестарта; битый JSON → лог + прежняя
копия, сервер не падает). Названия подставляются ТОЛЬКО при отображении
(fleet/device//print/CSV) и НИКОГДА не пишутся в БД/телеметрию — в данных только коды.

**Почему:**
- Дашборд без аутентификации → админ-UI редактирования справочника позволил бы
  любому в сети переименовывать организации; файл = модель доверия ingest_token.
- Render-time → переименование мгновенно отражается во всей истории и отчётах,
  ничего не мигрируя; клиент названий не получает (identity-labels: client never
  gets names).
- Код вне справочника → показываем код + чип «нет в справочнике» (ловит опечатки
  деплоя); телеметрия НЕ отклоняется (UNKNOWN over rejection).

**Унификация (один источник истины):** свободнотекстовая `devices.department`
(PATCH + группировка /print) DEPRECATED → `dept_code` + справочник через
COALESCE(имя по коду, department, 'Без отдела') на переходный период; edit-поле
device.html переключается на comment; колонка в БД остаётся (additive), из UI не
пишется. Связано: [[identity-labels-not-tenancy]].
