Активный тред (с 2026-06-24): объединить два раздела дашборда — «Топология» (`/topology`, подсистема netdisco, персистентный L2/L3-граф) и «Карта сети» (`/netmap`, аналитика `analytics/netmap.py`, эфемерные оверлеи) — в ОДНУ интерактивную «Карту сети». План v2 (10 фаз, R4): `docs/superpowers/plans/2026-06-24-network-map-unification-plan.md` (Прил.G = каталог OID/портов/адаптеров). Живой статус — `CONTINUITY.md`.

**Архитектура (решения владельца 2026-06-24, не пересматривать):**
- netdisco = ХРЕБЕТ (источник истины: узлы `net_devices`, реальные связи `net_links`, снимки/изменения/достижимость). netmap = ОВЕРЛЕИ (ICMP-качество, субсеть-аномалия, кластеры, анимации) — НЕ вторая модель.
- Идентичность = FK-связка, НЕ слияние таблиц. `net_devices` +nullable `device_id`/`printer_id`. Канонический ключ узла = `device_nid`. Три таблицы (devices/net_devices/printers) остаются — разные домены.
- Одна каноническая карточка на узел (`card_url`: agent>printer>net-infra). Один canvas-движок (Ф4). Контракт агента/`shared/schema.py` НЕ трогаем (нет bump CONTRACT_VERSION). Всё read-side (D7), RFC1918, под caps. Соблюдать `[[no-disabled-by-default]]` и `[[auto-merge-push-no-ask]]`.

**СДЕЛАНО (в origin/main):**
- **Ф1 ✅ MAC-FK хребет** — `[[netmap-identity-spine]]`. `net_devices +device_id/+printer_id`, `identity.link_identities`, `netmap.agent_mac_index`, связка в конце inventory-цикла; cleanup обнуляет FK (узел остаётся).
- **Ф2 ✅ единый ассемблер** — `[[netmap-unified-assembler]]`. `netdisco/unified.py::build_network_map(net_devices, net_links, snapshots, printers)` → суперсет `{nodes,links,subnets,totals}`. Хелперы `subnet_hint`/`quality_overlay`/`subnet_anomaly` вынесены в `analytics/netmap.py` (`build_netmap` byte-identical). Merge `21822bf`.
- **Ф3 ✅ единый API** — `[[netmap-unified-api]]`. `GET /api/v1/network-map/graph` → `build_network_map` через `GraphCache` (loader=`netdisco/cache.py::load_network_map` — DB fan-out get_net_devices/get_net_links/get_network_snapshots/get_printers→ассемблер); `/api/v1/netmap`+`/api/v1/topology/graph` → deprecated-алиасы (тот же граф, форма `nodes/links/subnets/totals`, БЕЗ `graph`-обёртки/`received_at`); `/topology/poll`+`/discovery/poll` инвалидируют кэш; `app.state.network_map_cache=GraphCache()` поднят в `create_app` (снят P11-LOW ленивый init). Merge `9c6ae9f`. Тесты `test_netmap_unified_api.py`(8).

**ДАЛЬШЕ (порядок 1→2→3→4→5; 6 после 4; 7,8 после 2; 9 после 7,8; 10 последняя):**
- **Ф4 (СЛЕДУЮЩАЯ, R3+review+XSS):** один canvas `web/templates/_netgraph.html` (на базе `topology.html`)+порт визуалов netmap (кластеры/качество/пакет-анимация/кольца-аномалии/пунктир wireless); глифы ВСЕХ типов (router/switch/ap/agent/printer/server/phone/endpoint/unknown); `/netmap` отдаёт единый граф (ассемблер+кэш); SSR-таблица+журнал+кнопка «собрать сейчас». E2E headless: типы/wireless/поиск/зум/XSS-инертен/reduced-motion.
- **Ф5 (R3+review+XSS):** панель управления — фильтры (источник/тип/статус/подсеть/VLAN/medium/confidence/agent-presence + «скрыть ARP-only»), тоггл слоёв, раскладки+персист позиций (localStorage по nid), боковая панель на клик, сохранить-вид, экспорт PNG/CSV/JSON; ADVANCED (опц.): изоляция N-hop, путь BFS (`graph.py`), первопричина (`correlation.py`), машина времени по `net_topology_snapshots`.
- **Ф6 (R3+review):** одна каноническая карточка — резолвер `card_url` (уже в узле ассемблера), секция «Сеть» в `device.html`/`printer_detail.html` (общий partial `_net_section.html` из `net_device.html`), `/netdisco/device/{nid}` связанного → 302 на каноническую.
- **Ф7 (R4+security-review):** Tier-1 SNMP-углубление — LLDP port-id/mgmt-addr, dot1q VLAN-FDB, ifXTable, LLDP-MED, ENTITY, dot1dStp, PoE, HOST-RES, ipNetToPhysical, РЕАЛЬНЫЕ wireless-рёбра client→AP (AIRESPACE/Aruba/MikroTik). Пишет аддитивно `net_links.a_port/b_port/vlan/medium`, `net_devices.subtype`, `net_interfaces.if_alias`.
- **Ф8 (R4+security-review):** Tier-2 пассив — reverse-DNS/mDNS/SSDP/NetBIOS/WSD/TLS-баннер; де-аноним «Неизвестного». Новые `netdisco/passive.py`+`naming.py`. p0f SKIP (Win raw-socket).
- **Ф9 (R4+security-review):** Tier-3 опц-адаптеры — `NetworkAdapter` ABC + `adapter_merge.py` (merge-by-MAC) + MikroTik/UniFi/Redfish/NetFlow (dep `netflow` PyPI — одобрено, единственная не-stdlib, изолирована); креды через DPAPI (`netdisco/credentials.py`); DNA/Meraki/OPNsense documented-only.
- **Ф10 (R2+финальный review):** демонтаж «Топологии» — убрать nav+route, `/topology`→301 на `/netmap`, верифицировать каждую строку матриц плана в `/netmap`, мигрировать `test_topology_web.py`→`test_netmap_web.py`.

**Карта кода (для продолжения):** ассемблер `server/netdisco/unified.py`; хелперы-оверлеи `server/analytics/netmap.py` (`subnet_hint`/`quality_overlay`/`subnet_anomaly`/`agent_mac_index`/`build_netmap`/`subnet_context_for`); идентичность `server/netdisco/identity.py::device_nid`(`nd-chassis/-sn/-mac/-ip/-unknown`)/`link_identities`; снимок-сериализатор `server/netdisco/reconcile.py::_graph` (форма узла `{nid,dev_type,ip,hostname}` + ребра `{a,b,via_source,confidence,link_kind,ambiguous}`); кэш `server/netdisco/cache.py::GraphCache(ttl_sec,loader,clock)`; граф-операции `server/netdisco/graph.py` (`build_graph`/`reachable_from`/`path_to_root`/`find_root_cause`); БД-ридеры `server/db.py`: `get_net_devices`(+device_id/printer_id), `get_net_links`(a_nid/b_nid/a_if/b_if/link_kind/via_source/confidence), `get_network_snapshots`(adapters/neighbors/quality), `get_printers`(printer_id/ip/mac/...), `get_latest_topology_snapshot`/`store_topology_snapshot`; роуты `server/web/dashboard.py` `/netmap`·`/topology`·`/netdisco/device/{nid}`; API `server/api.py::topology_graph`. Шаблоны `web/templates/netmap.html`+`topology.html` (сольются в `_netgraph.html` Ф4).
