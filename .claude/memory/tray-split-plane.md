# Трей = отдельный процесс в сессии пользователя («две плоскости»)

**Решение (2026-06-12, спека tray-client):** UI-клиент `srp-tray.exe` живёт в сессии
каждого пользователя (HKLM\...\Run), агент остаётся SYSTEM scheduled task. IPC —
ТОЛЬКО односторонний `status.json` (агент пишет атомарно tmp+`os.replace`, трей
читает; свежесть < 15 мин = жив). Канала команд трей→агент НЕТ.

**Почему (не очевидно):**
- SYSTEM-процесс физически не видит личный сертификат пользователя:
  `Cert:\CurrentUser\My` под SYSTEM = хранилище самой SYSTEM (поэтому W1.2-сборщик
  сертификатов этот случай не покрывает). Проверка личного серта возможна только
  из сессии пользователя.
- Из session 0 нет UI (трей/уведомления не доставляются by design).
- Нет канала команд → нет поверхности атаки/валидации; «собери сейчас» не нужен
  при heartbeat 5 мин.

**Как применять:** любые новые «пользовательские» данные (HKCU, профиль, серты
пользователя) собирает трей, не агент; любые новые «машинные» данные — агент.
status.json никогда не содержит секретов (ingest_token/hash — пин-тест).
