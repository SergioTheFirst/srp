# Сертификаты: группировка по Subject + окна 14дн/4ч и 7дн/1×день

**Решение (2026-06-12, спека tray-client §3):** напоминания об истечении личного
сертификата группируются по нормализованному Subject CN. Продление ЭЦП = НОВЫЙ
сертификат того же субъекта → валидный преемник ГАСИТ все напоминания о старом
(до и после истечения) + однократный инфо-балун «обнаружен новый сертификат».
Разные субъекты независимы (разные роли/системы).

**Окна (директива пользователя):** валиден >14 дн → тишина; 8–14 дн → балун каждые
4 ч (жёлтый); ≤7 дн → каждые 4 ч (красный); ИСТЁК → ровно 1 балун в КАЛЕНДАРНЫЙ
день × 7 дней, затем балуны замолкают навсегда (красная иконка и панель остаются).
14 дн — правка пользователя при утверждении (в v2 было 30).

**Почему:** наивный клиент пилит пользователя за уже заменённый сертификат — главный
источник недоверия к таким нотификаторам; subject-группировка убирает этот шум, не
маскируя истечение сертификатов других ролей. После истечения шум бесконечных
балунов не помогает — 7 дней достаточно, дальше сигнал живёт в иконке/панели.

**Как применять:** движок = чистая `should_nag(now, group, state)`; анти-спам
персистентен (%LOCALAPPDATA%\SRP\tray_state.json, по thumbprint; «раз в день» = по
календарной дате, не 24 ч). Даты из PS только epoch-числами.
