Байесовский риск по классам отказа считается в лог-оддсах:
`posterior = sigmoid(prior + Σ весов признаков)` (`server/scoring/bayesian.py`). Каждый
член подписан → результат объясним по построению (дашборд показывает вклад признаков).

**Why:** веса заданы **вручную и не калиброваны** — это заглушка до появления
размеченных отказов. Абсолютное значение вероятности доверия не заслуживает; ценность
сейчас — ранжирование машин и объяснимость, а не точная вероятность.

**How to apply:** не выдавать число за откалиброванную вероятность. Конечная цель —
survival-модель + изотоническая калибровка на реальных отказах. Сохраняй
подписанность каждого члена (объяснимость — ключевое свойство, не ломать).
